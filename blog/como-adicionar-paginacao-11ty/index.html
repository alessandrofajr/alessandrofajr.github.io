Como adicionar paginação a um blog no 11ty (eleventy)

2024-07-27

Com a seção do nosso blog pronta, a tendência é que a lista a lista de publicações fique extensa com o passar do tempo. O 11ty tem uma solução de paginação integrada que é muito simples de implementar.

  1. Volte ao arquivo blog/index.md e altere o front matter:
---
layout: default
title: blog
pagination:
  data: collections.blog
  size: 10
  reverse: true
  alias: posts
---

Ao incluir a especificação de pagination, apontamos para uma coleção de dados do blog, o tamanho da página, definimos que queremos ordenar essa coleção de trás para frente e um sinônimo.

  1. É preciso alterar outro trecho do blog/index.md, modificando o loop que criamos anteriormente, pedindo para que o código recupere cada post a partir dos itens de paginação. Além disso, adicione links para permitir a navegação. O arquivo completo ficará assim:
---
layout: default
title: blog
pagination:
  data: collections.blog
  size: 10
  reverse: true
  alias: posts
---

## Blog
<ul>
{% for post in pagination.items %}
<li><a href="">{{ post.data.title }}</a></li>
{% endfor %}
</ul>

<nav>
  {% if pagination.href.previous %}
    <a href="">Página anterior</a>
  {% else %}
  {% endif %}
  {% if pagination.href.next %}
    <a href="">Próxima página</a>
  {% else %}
  {% endif %}
</nav>

O código dentro das tags <nav> possuem uma condicional para exibir os links de "próxima página" e "página anterior" somente se esses links existirem. Dessa forma, evitamos de criar um link que não leva para lugar nenhum.

Crie alguns posts para testar a paginação e brinque com o tamanho da paginação. Dois passos bem simples.
